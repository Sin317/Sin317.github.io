Git Commands, Commit History, and Merge Mechanism#

Git is a powerful distributed version control system that allows developers to track changes, collaborate, and manage codebases efficiently. This document provides a comprehensive guide to Git commands, explains how Git stores commit history, and describes the merging process.

Essential Git Commands#

Initialize a new Git repository#

git init

Configure user details for commits#

git config –global user.name “Your Name”
git config –global user.email “your.email@example.com”

Check Git configuration#

git config –list

Clone a remote repository#

git clone

Create a new repository and connect it to a remote#

git init
git remote add origin

Check the current status of your repository#

git status

Stage files for commit#

git add # Add a specific file
git add . # Add all changes

Commit staged changes with a message#

git commit -m “Descriptive commit message”

List all branches#

git branch

Create a new branch#

git branch

Switch to a branch#

git checkout

Create and switch to a new branch#

git checkout -b

Delete a branch locally#

git branch -d

Push changes to a remote branch#

git push origin

Pull the latest changes from the remote branch#

git pull origin

Fetch updates without merging#

git fetch

Merge a branch into the current branch#

git merge

Resolve merge conflicts manually and mark them resolved#

git add
git commit -m “Resolved merge conflicts”

Revert to the last committed state#

git checkout –

Unstage a file#

git reset

Reset to a previous commit (careful: destructive)#

git reset –hard

How Git Stores Commit History#

Git stores its commit history in a structure called a Directed Acyclic Graph (DAG). Here’s how it works:

1. Commit Objects#

Each commit in Git is represented as an object containing:

A unique SHA-1 hash (identifier). Metadata: Author, timestamp, and commit message. A reference to its parent commit(s).

2. Branch Pointers#

Branches are lightweight movable pointers to commits. The default branch is often named main or master. When you commit, Git automatically moves the branch pointer to the latest commit.

3. HEAD Pointer#

HEAD is a special pointer that indicates the current branch or commit you’re working on.

Example of a simplified commit history: A – B – C (main)
D – E (feature)

n this case:

HEAD points to feature. main branch ends at commit C.

How Git Merges Branches#

1. Fast-Forward Merge#

Occurs when the branch to be merged has no divergence from the main branch.

Git simply moves the branch pointer forward.

Before:
A – B – C (main)
D – E (feature)

After:
A – B – C – D – E (main, feature)

2. Recursive (Three-Way) Merge#

Occurs when both branches have new commits. Git combines the changes from the latest common ancestor into a new commit.

Before:
A – B – C (main)
D – E (feature)

After:
A – B – C —– F (main) \ / D – E (feature)

3. Merge Conflicts#

Conflicts occur when changes in the same part of a file exist on both branches. Git will stop the merge and mark the conflict in the file. Conflict markers look like this:

<<<<<<< HEAD
Code from the current branch
=======
Code from the branch being merged
>>>>>>> branch-name
